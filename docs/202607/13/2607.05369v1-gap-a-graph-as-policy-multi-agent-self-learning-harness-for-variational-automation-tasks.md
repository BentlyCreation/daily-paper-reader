## 摘要
为了使机器人在商业和工业应用中可靠工作，智能体编码系统的最新进展能否将可解释的机器人编程与无模型策略的开放世界适应性结合起来？我们聚焦于“变分自动化”（VA），这是一类比固定自动化在物体几何和姿态上具有更大变化的任务。对于必须在商业和工业应用中持续可靠执行的VA任务，无模型策略往往难以缩小可靠性差距。受任务与运动规划（TAMP）和机器人操作系统（ROS）先前工作的启发，我们提出了图策略（GaP），这是一个多智能体编码框架，能从模块化开放机器人技能库（MORSL）生成包含感知、规划和控制节点的有向计算图。然后GaP生成内部仿真环境，并行排练不同图的任务实例，以迭代优化图结构和参数，提高成功率和吞吐量。在8个新的开放VA任务基准（4个仿真、4个真实世界）上的评估表明，GaP能达到显著优于基线的成功率。详情、代码和数据见：https://graph-robots.github.io/gap

## Abstract
For robots to work reliably in commercial and industrial applications, can recent advances in agentic coding systems combine interpretable robot programming with the open-world adaptability of model-free policies? We focus on "Variational Automation" (VA), a class of tasks that have larger variations in object geometry and pose than fixed automation. Model-free policies often struggle to close the reliability gap for VA tasks, which must be executed persistently and reliably in commercial and industrial applications. Motivated by prior work on Task and Motion Planning (TAMP) and the Robot Operating System (ROS), we introduce Graph-as-Policy (GaP), a multi-agent coding harness that generates directed computation graphs with perception, planning, and control nodes from a Modular Open Robot Skill Library (MORSL). GaP then generates an internal simulation environment to rehearse task instances with different graphs in parallel to iteratively refine the graph structure and parameters to improve success rates and throughput. Evaluation with 8 new open VA task benchmarks, 4 in-simulation and 4 in real-world, suggests that GaP can achieve success rates that significantly outperform baselines. Details, code, and data can be found online: https://graph-robots.github.io/gap

## 2. 方法论：核心思想、关键技术细节

*   **核心思想**：GaP将机器人策略表示为一个有向计算图（Directed Computation Graph），图中节点是来自模块化开放机器人技能库（MORSL）的原子功能单元（如感知、规划、运动控制），边则表示数据流和执行顺序。通过多智能体编码框架自动生成和优化该图。
*   **关键技术细节**：
    1.  **模块化开放机器人技能库 (MORSL)**：包含51个初始技能，涵盖感知（SAM2/3、Grounding DINO）、抓取规划（GraspGen）、运动规划（cuRobo）等。每个技能通过标准的元数据声明输入、输出和前提条件，便于智能体调用和组合。
    2.  **多智能体生成框架**：由编排代理（Orchestration Agent）将VA任务描述分解为语义片段，并分配给相应的技能代理（Skill Agents）。技能代理负责为各自片段生成局部的功能性子图，编排代理再将这些子图聚合连线成完整的可执行计算图。该结构通过分离图生成与测试，限制了单个代理的上下文窗口，降低了作弊和幻觉风险。
    3.  **自我学习（Self-Learning）**：通过内部仿真（NVIDIA Isaac）进行“排练”。算法流程如下：
        *   **初始图生成**：根据语言描述生成初始计算图G0。
        *   **场景变分采样**：从任务任务信念空间B中采样N个并行任务实例。
        *   **并行排练**：在当前图Gj-1下对所有实例进行回放，记录成败。
        *   **分析失败与图优化**：分析物理执行数据（如姿态、接触），定位失败根因（如几何碰撞、抓取偏移）。LLM代理据此对图进行迭代更新，包括替换节点、修改参数等，直至性能（成功率与吞吐量的加权组合）达到平稳。
    4.  **策略表示**：`G* = argmax_G E_{xi~B}[J(π(a|I, G))]`，优化目标是在信念空间上最大化奖励函数J（包括成功率和吞吐量）。

## 6. 主要结论与发现

*   GaP在8个VA基准任务上显著优于所有基线（包括π0.5, MolmoAct2, TiPToP, CaP-X），在极具挑战性的变体中仍能保持>0.9的成功率。
*   自我学习（内部仿真排练）能有效提升初始图性能：例如，爆米花任务从33%提升至94%（仿真），并成功迁移到真实环境（90%）。
*   GaP可以与现有VLA模型结合（GaP+VLA），通过引导机器人末端到目标上方，大幅提升VLA在位置变化下的成功率（提升超过两倍）。
*   图结构是GaP的关键：移除图结构（改用单智能体生成原始代码）导致成功率降为0。多智能体分工和图验证也至关重要。

## 7. 优点

*   **结构化与可解释性**：计算图作为策略表示，清晰定义了感知、规划、控制的模块与逻辑流，支持模块复用、推理和故障定位。
*   **多智能体协同与自学习**：通过编排代理和技能代理的层次化协作，有效利用LLM能力；闭环的仿真“排练-反馈-优化”机制自动提升策略可靠性。
*   **鲁棒性与泛化能力**：在物体位置、姿态、类别等多种变体下表现稳健；能从仿真成功迁移到真实场景。
*   **集成能力**：可与ROS节点/技能库无缝集成（如电缆插入任务），融合经典工程方法与学习范式；可与VLA模型结合，取长补短。

## 8. 不足与局限

*   **执行速度**：GaP的执行时间（如单次抓放约67秒，插线约30秒）远低于工业标准（7秒/次），主要瓶颈在于VLM推理、运动规划时间。
*   **任务范围**：基准任务主要集中在准静态抓放操作，仅电缆插入涉及力传感。对可变形物体、动态力、移动目标等复杂场景的适用性尚未验证。
*   **可靠性天花板**：尽管显著优于基线，但执行可靠性尚未达到工业级水平（如真实“爆米花”任务仍有2/20失败）。自我学习的效果依赖于高质量仿真反馈和参数调优。
*   **假设条件**：VA设定假设工作空间、机器人、传感器和物体范围已知，这限制了其在完全陌生环境中的泛化能力，但属于任务类别的固有属性。
