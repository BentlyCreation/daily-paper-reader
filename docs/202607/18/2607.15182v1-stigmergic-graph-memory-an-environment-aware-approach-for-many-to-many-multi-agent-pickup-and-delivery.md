## 摘要
自动化履行仓库必须持续分配和执行取送货工作，同时避免拥堵。在多对多式多智能体取送货（MAPD）中，请求指定一个库存单位而非固定端点，要求控制器在路径规划前选择智能体、源点和目标点。现有的图引导方法主要在目标确定后影响路径规划，而端点实例化不受近期交通状况的影响。我们引入了启发性图记忆（SGM），这是一种有界、衰减的记忆层，记录仓库节点和有向边上的近期执行信号，用于对可行端点和路径偏好进行排序，而不改变碰撞约束或规划器有效性。在五种布局、三种负载水平、每种条件下25个种子的配对请求流中，SGM在所有15种地图-负载条件下优于两种重建的多对多分配基线，配对吞吐量提升20.5-36.7%。这些结果表明，近期执行记忆可以通过塑造哪些可行目标进入规划器（而不仅仅是智能体如何前往已固定目标）来提高仓库吞吐量。

## Abstract
Automated fulfillment warehouses must continuously assign and execute pickup-and-delivery work while avoiding congestion. In many-to-many Multi-Agent Pickup and Delivery (MAPD), a request specifies a stock-keeping unit rather than fixed endpoints, requiring the controller to select an agent, source, and destination before path planning. Existing graph-guidance methods primarily influence routing after goals are fixed, leaving endpoint instantiation uninformed by recent traffic. We introduce Stigmergic Graph Memory (SGM), a bounded, decaying memory layer that records recent execution signals on warehouse nodes and directed edges to rank feasible endpoints and route preferences without altering collision constraints or planner validity. Across paired request streams on five layouts, three load levels, and 25 seeds per condition, SGM outperforms two reconstructed many-to-many allocation baselines in all 15 map-load conditions, with paired throughput gains of 20.5-36.7%. These results show that recent execution memory can improve warehouse throughput by shaping which feasible goals enter the planner, not only how agents travel to already fixed goals.

## 2. 论文提出的方法论

- **核心思想**：受启发性（stigmergy）协作启发，提出Stigmergic Graph Memory (SGM)，一个在仓库节点和有向边上维护的、有界衰减的记忆层。该记忆记录不同类型的近期执行事件，并分别用于端点评分（Endpoint Steering）和路由引导（Route Guidance），而不改变碰撞约束或规划器有效性。
- **关键技术细节**：
  - **记忆更新**：对每个通道 \(c\) 和图形元素 \(x\)，记忆值按 \(M_{t+1}^c(x) = \rho_c M_t^c(x) + \Delta_t^c(x)\) 演化，其中 \(\rho_c\) 是通道特定的衰减因子（如等待/延迟0.85，阻塞/端点压力0.90等）。
  - **端点评分**：对每个可行候选 \((a_i, \tau_n, s, d)\)，SGM在M2M风格的基础成本 \(C_0\) 上加上记忆路径惩罚：\(C = C_0 + \lambda_p P_{\text{path}}(a_i, s, d)\)，其中 \(P_{\text{path}}\) 是沿单位成本最短路径的当前有向边惩罚之和。
  - **路由引导**：将有向边成本转换为有界正值：\(\tilde{c}_t(u,v) = 1 + \min\{\kappa, \lambda_r R_t(u,v)\}\)，其中 \(R_t\) 聚合归一化的拥塞、阻塞、延迟、逆流、成功流以及计划占用成本。所有合法边成本保持在 \([1, 1+\kappa]\) 之间，确保无负成本或零成本循环。
  - **规划器**：采用滚动视野碰撞解决（RHCR）结合优先级搜索（PBS）作为底层多智能体路径规划（MAPF）后端。
  - **控制器变体**：SGM（连续使用记忆）、ASGM（自适应，仅在强拥塞证据下激活记忆）、M2M/M2M-wSKU（静止分配，无记忆）。

## 6. 论文的主要结论与发现

- **SGM在所有15个地图-车队条件下均优于M2M和M2M-wSKU**，配对吞吐量提升20.5–36.7%（最终完成任务数）。
- **端点引导是吞吐量提升的主要来源**：端点仅消融保留了99.3–100.5%的完整SGM吞吐量，而路由仅消融仅达到78.5%左右，说明路由引导单独无法恢复因端点选择不佳造成的性能损失。
- **路由引导改善执行效率**：完整SGM相比端点仅，减少了规划器时间（–9.1%）、等待事件、阻塞移动和阻塞移动重规划，表明路由引导降低了执行代价而不牺牲吞吐量。
- **ASGM提供了计算-吞吐量权衡**：在高负载下具有最佳的归一化吞吐量（每候选评估/每秒计算的任务数）。
- **路由控制实验**表明：仅添加近期交通路由或静态高速公路到M2M-wSKU后吞吐量变化很小（–0.1%和–1.4%），而SGM提升约29%，证明效果来自于端点选择而非路由。
- **转移研究**：在中等规模布局上SGM仍比M2M-wSKU提升7.3%完成任务数，表明方法泛化性良好。
