## 摘要
智能体间通信对于在部分可观测条件下协调多智能体强化学习智能体以在合作游戏中有效执行至关重要；然而，现实世界中的带宽限制要求稀疏交互。先前的方法主要通过优化信息论代理来解决这一权衡。我们认为这些统计代理与真实目标根本不一致：一条消息可能信息量很大，但与任务的联合回报无关。在这项工作中，我们提出了面向目标效率的消息遗忘（MUTE）框架，该框架将通信缩减视为一个价值引导的机器遗忘问题。MUTE使用基于注意力的估计器严格量化反事实消息价值，并系统性地遗忘从无任何通信约束训练的策略中低价值消息的传输。这是通过一个双目标机制实现的，该机制在保持原始联合策略回报的同时强制执行通信稀疏性。我们推导了由这种稀疏化引起的性能差距的理论上界，保证了受控的回报退化。我们还对各种复杂多智能体环境进行了MUTE的实证评估，实现了80%到90%的带宽缩减，同时保持了与最先进基线相当的性能。

## Abstract
Inter-agent communication is critical for coordinating Multi-Agent Reinforcement Learning (MARL) agents under partial observability to perform effectively in cooperative games; however, real-world bandwidth constraints demand sparse interactions. Prior approaches primarily address this trade-off by optimizing information-theoretic surrogates. We argue that these statistical proxies are fundamentally misaligned with the true objective: a message can be highly informative yet irrelevant to the joint return of the task. In this work, we propose Message Unlearning for Targeted Efficiency (MUTE), a framework that views communication reduction as a value-guided machine unlearning problem. MUTE rigorously quantifies the Counterfactual Message Value using an attention-based estimator, and systematically unlearns the transmission of low-value messages from a policy trained without any communication constraints. This is achieved through a dual-objective mechanism that enforces communication sparsity while preserving the return of the original joint policy. We derive a theoretical upper bound on the performance gap induced by this sparsification, guaranteeing controlled return degradation. We also empirically evaluate MUTE on various complex multi-agent environments, achieving 80% to 90% bandwidth reduction while maintaining performance comparable to state-of-the-art baselines.

## 2. 方法论：核心思想、关键技术细节、公式或算法流程
### 核心思想：MUTE（Message Unlearning for Targeted Efficiency）
- **两阶段流程**：
  1. **预训练阶段**：先使用标准MARL（基于MASIA框架）训练一个全通信专家策略 \( \pi^* \) 和价值函数 \( Q_{\text{tot}}^* \)。
  2. **遗忘阶段**：通过训练一个**消息价值估计器（MVE）** 来估计每条消息的“反事实消息价值（CMV）”，然后对低价值消息施加稀疏性惩罚（L1范数），同时用**行为锚定损失**来稳定策略，防止性能崩溃。

### 关键技术细节
- **反事实消息价值（CMV）**：定义为消息 \( m_i \) 存在与不存在时联合Q值的差异。公式：
  \[
  \Delta Q_i(\tau, m) = Q_{\text{tot}}^*(\tau, a) - Q_{\text{tot}}^*(\tau, \tilde{a})
  \]
  其中 \( \tilde{a} \) 是在屏蔽 \( m_i \) 后重新采样队友动作得到的联合动作。
- **消息价值估计器（MVE）**：使用**自注意力机制**架构，以所有智能体的消息集合作为输入，同时输出所有消息的CMV估计。选择自注意力的原因是：CMV需满足置换不变性，且具有上下文依赖性（例如重复消息时单条消息边际价值低）。
- **双目标损失**：
  - **稀疏损失**：\( L_{\text{sparse}}(\psi) = \sum_{m \in M_{\text{red}}} \| m \|_1 \)，其中 \( M_{\text{red}} \) 是CMV估计低于自适应阈值的消息集合（ \( \lambda \mu_{\text{MVE}} \) ）。
  - **行为锚定损失**：\( L_{\text{anchor}}(\theta, \psi) = \sum_{i=1}^N \| Q_i^*(\tau_i, a_i, m_{-i}) - Q_i(\tau_i, a_i, m_{-i}; \theta) \|_2^2 \)，使当前策略的个体Q值接近冻结的基线策略。
  - **最终损失**：\( L = L_{\text{sparse}} + \beta L_{\text{anchor}} \)。
- **理论保证**：推导了遗忘后性能差距的上界（Theorem 4.2），确保性能退化可控（依赖于MVE误差δ和Q值近似误差ϵ的有界性）。

### 算法流程（文字说明）
1. **阶段1：预训练**（2M步）：训练无限制通信的专家策略 \( \pi^* \) 和 \( Q_{\text{tot}}^* \)。
2. **阶段2：MVE训练**（0.5M步）：利用专家策略收集轨迹，通过反事实重采样生成CMV标签，训练注意力MVE网络。
3. **阶段3：值引导通信遗忘**（1.5M步）：固定MVE，使用双目标损失更新策略和消息生成器，逐步压制低价值消息的生成。

## 6. 主要结论与发现
- **通信效率**：MUTE在10个基准环境中实现了80%~99%的带宽缩减，同时性能与全通信基线MASIA相当或接近，显著优于其他通信缩减方法。
- **选择性**：MUTE保留的消息是真正对回报关键的：在消息丢弃率实验中，MUTE的抗丢弃曲线（AUC）在所有环境中最高，说明其识别的冗余消息正确。
- **机制必要性**：行为锚定和MVE都是必需的。移除锚定会导致策略崩溃；随机选择消息（w/o MVE）会严重降低性能。
- **理论验证**：实验追踪了Q值误差和MVE误差，证实了理论假设（误差有界）在实际中成立。
- **局限性**：MUTE的CMV只估计单条消息的边际价值，可能无法捕获多条消息之间的高阶交互。论文通过行为锚定损失一定程度上缓解，但扩展至子集感知估计是未来方向。

## 7. 优点：方法或实验设计上的亮点
- **方法创新**：首次将通信缩减视为**价值引导的机器遗忘**，直接优化任务回报而非信息论代理，思路新颖。
- **理论贡献**：推导了遗忘后性能差距的上界，为算法提供可靠性保证，且在实际中验证了误差有界。
- **实验设计**：
  - 包含多种类型环境（诊断、交通、星际微操），覆盖面广。
  - 设计消息丢弃率实验（dropout），直接测试保留消息的价值，结果非常有说服力。
  - 消融实验全面，验证了每个关键组件的必要性。
- **计算效率**：相比基于Shapley值的SMS（指数复杂度），MUTE的MVE通过注意力机制实现单趟所有消息的估计，计算效率更高。

## 8. 不足与局限
- **高阶交互问题**：CMV仅考虑单条消息的边际贡献，当多条消息存在协同或冗余时，可能错误标记。虽然行为锚定会提供反馈，但理论上不如子集感知方法严密。
- **依赖专家策略**：MUTE需要先训练一个无限制通信的专家策略，这本身可能时间成本高，且在部分任务中（如SMACv2 Terran 10v10），由于环境随机性，全通信策略优势不大，其他非通信方法能取得更好性能。
- **实验比较范围**：未与部分最新通信缩减方法（如NDQ、IMAC的直接对比）在相同配置下严格复现；在SMACv2上与非通信方法MAPPO相比，MUTE性能略低（但MAPPO需要更多训练步数且通信代价为0）。
- **超参数敏感性**：λ和β需要调优，论文虽然进行了消融，但对新环境可能需要重新调整。
- **可扩展性**：在大规模智能体场景（如20+）中，MVE的计算和性能可能会下降（Table 13(b)中20个智能体时胜率下降至62.5%），未在更大规模环境上测试。
- **应用限制**：假设通信为广播模式，若为点对点通信则需修改消息价值定义。
