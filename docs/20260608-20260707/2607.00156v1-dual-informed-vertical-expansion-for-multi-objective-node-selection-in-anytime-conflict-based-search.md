## 摘要
冲突基搜索（CBS）是多智能体路径规划（MAPF）领域领先的精确算法，但其高层节点选择规则通常被视为固定的实现细节。标准的最佳优先搜索在最小化展开节点和闭合最优性证明方面表现强劲，但可能维持较大的前沿集、中断父子扩展序列，并且在终止前无法提供可行的当前最优解。本文研究节点选择作为精确CBS的一级设计选择。我们引入双知情垂直扩展（DIVE），这是一种在潜行之间采用最佳边界、在潜行内部采用深度导向的策略。DIVE每次从当前最佳边界前沿开始潜行，沿着有希望的子节点利用父子局部性，并使用当前最优剪枝来限制无益的离题。我们通过分支定界视角形式化CBS节点选择，证明遍历策略可以在不影响精确性的情况下改变，并分析展开节点数、潜行中断次数、队列大小以及原始对偶边界进展之间的权衡。分析预测了三种互补的极端情况：最佳优先搜索节点高效，迭代加深内存高效，而DIVE在保持常规最佳边界重锚的同时潜行高效。在标准MAPF基准上的实验支持这一权衡图谱。DIVE一致地减少了潜行中断次数，提供了具有认证间隙的早期当前最优解，使用的队列内存远少于最佳优先搜索，并在密集或内存受限场景中受益于热启动和简单的响应变体。

## Abstract
Conflict-Based Search (CBS) is a leading exact algorithm for Multi-Agent Path Finding (MAPF), but its high-level node-selection rule is usually treated as a fixed implementation detail. Standard best-first selection is strong for minimizing expanded nodes and closing the optimality certificate, yet it can maintain a large frontier, interrupt parent-child expansion sequences, and provide no feasible incumbent until termination. This paper studies node selection as a first-class design choice for exact CBS. We introduce Dual-Informed Vertical Expansion (DIVE), a policy that is best-bound between dives and depth-oriented within a dive. DIVE starts each dive from the current best-bound frontier, follows promising children to exploit parent-child locality, and uses incumbent pruning to limit unproductive excursions. We formalize CBS node selection through a branch-and-bound view, prove that the traversal policy can be changed without affecting exactness, and analyze the resulting trade-offs among expanded nodes, dive breaks, queue size, and primal-dual bound progress. The analysis predicts three complementary extremes. Best-first search is node efficient, iterative deepening is memory efficient, and DIVE is dive efficient while retaining regular best-bound reanchoring. Experiments on standard MAPF benchmarks support this trade-off map. DIVE consistently reduces dive breaks, provides early incumbents with certified gaps, uses substantially less queue memory than best-first search, and benefits from warm starts and simple responsive variants in dense or memory-limited regimes.

### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：Dual-Informed Vertical Expansion（DIVE）是一种“潜行之间最佳边界，潜行之内深度优先”的混合节点选择策略。每次潜行从当前全局最佳边界的开放节点开始，然后沿着最有希望的子节点（即最低下界）向下探索，利用父子局部性（子节点只增加一条约束，只需重规划一个智能体）。当潜行变得无益（子节点下界 ≥ 当前最优解值）或到达可行解时，停止潜行，并重新从全局最佳边界开始下一次潜行。  
- **关键技术细节**：  
  - **当前最优解剪枝（Incumbent Cutoff）**：如果开放节点的下界 ℓ(n) ≥ 当前最优值 zP，则该节点及其所有后代无法改进当前最优解，可以被安全剪枝。该机制限制无益的深度探索。  
  - **多样化重锚（Diversified Reanchoring）**：每次潜行结束后，下一次潜行的根节点由全局规则（最小 ℓ 值）从所有开放节点中选择，避免像纯深度优先搜索那样陷入某个无益分支。  
  - **保持常规证明进展**：由于每次潜行后都回到最佳边界，DIVE 能持续提高对偶边界（下界），从而保证最优性证明的推进。  
- **算法流程（文字描述）**：  
  1. 初始化开放队列 OPEN = {根节点}，设置当前最优解 zP = ∞（或热启动值）。  
  2. 若 OPEN 非空，执行以下循环：  
     - 根据上一扩展节点 n_last 的子节点情况选择下一步节点：  
       - 若 n_last 存在开放子节点且最低下界子节点 ℓ(cmin) < zP，则选择该子节点继续潜行。  
       - 否则，选择 OPEN 中 ℓ 最小的节点（最佳边界）作为新的潜行根节点。  
     - 从 OPEN 中移除选中节点，扩展它：若其联合计划无冲突，更新当前最优解和最优值；若有冲突，生成两个子节点（各禁止一方冲突约束），将满足 ℓ(c) < zP 的子节点加入 OPEN。  
     - 同时检查下界剪枝条件。  
  3. 返回当前最优解（若为空说明无可行解）。  
- **理论证明**：将 CBS 建模为分支定界（B&B）算法，证明只要下界是允许的、分支覆盖所有可行解，节点选择策略的改变不影响算法正确性。DIVE 的下界剪枝也是有效的，因此是精确的。

### 6. 论文的主要结论与发现

- **节点选择是 CBS 的一级设计维度**，不同策略在展开节点数、潜行连续性、队列内存和可行解发现之间存在根本性权衡，不存在单一最优策略。  
- **DIVE 占据独特的帕累托点**：  
  - 相比 BFS：潜行中断减少 6%–51%，最大队列大小减少 44%–86%，同时能提供早期可行解和认证间隙。  
  - 相比 ID：展开节点数远少于 ID（DIVE 比 BFS 多 18%–57%，而 ID 多 13%–134%），潜行中断数更少，队列大小仍比 BFS 小很多但比 ID 大（ID 队列几乎恒定极低）。  
- **DIVE 的额外节点开销来自于受控的原始区域探索**（最优区域和原始区域），而非重复重启或丧失对偶边界。  
- **DIVE 具备任意时间能力**：几乎总是能在第一个潜行中找到可行解（平均只需一次潜行中断），且带有认证间隙；在 BFS 完成所需节点预算时，DIVE 已有 46%–100% 的实例找到最优解并给出间隙不超过 0.6% 的当前最优解。  
- **热启动大幅提升密集场景性能**：在 `test-5-5` 33% 占据率下，无热启动 DIVE 全部超时，加上 PIBT 热启动后解出 35/40 个实例，与 BFS 持平，优于 ID。  
- **多样化重锚是关键机制**：仅使用当前最优剪枝而无多样化重锚的 DFS 策略，在节点展开和潜行中断上均显著劣于 DIVE，证实了“每次潜行后回到最佳边界”的必要性。  
- **响应式变体 MC-DIVE** 能够在维护较低队列内存的同时，避免过度保守的深度优先行为，展示了闭环节点选择的潜力。

1. **实验覆盖广度有限**：所有实验在单一硬件条件下（单线程笔记本）进行，未在分布式或高性能集群上验证；仅测试了 5 种地图，未包含更大或更复杂的现实世界地图（如仓库大规模实例）。
